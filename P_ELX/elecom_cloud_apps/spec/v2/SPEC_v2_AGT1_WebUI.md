# AdminLink Requirements Specification

## Client Request

**EN:** Client wants to use the AdminLink service.  

---

## Glossary

| Term | Definition |
|------|-----------|
| AdminLink | The cloud-based device management service |
| Agent | The on-device software component that communicates with AdminLink |
| Device ID | UUID v4 generated by the device; used as identifier in all API calls |
| Registration Code | A 9-digit hex code generated for device registration (`register code` / `regist_cd`) |
| dev_id_changed | Flag (0 or 1) in API response; 1 indicates the device has been factory-reset after prior registration |
| agt_daily_sec | Seconds offset from 22:00:00 for daily processing execution time |
| agt_upload_sec | Seconds offset from :00:00 for hourly status upload timing |
| tmp_reg_expiry | Temporary registration expiry datetime (Zero-touch flow only) |
| re-registerable | State where a device was previously registered, factory-reset, and can be re-registered |
| Periodic Notification | `sts_type = 0` — status update sent by scheduled hourly process |
| Event Notification | `sts_type = 1` — status update triggered by an event |
| Optional Notification | `sts_type = 2` — status update triggered manually |

## Device Registration State Machine

| State | Description | Triggered By | Transitions To |
|-------|-------------|-------------|----------------|
| `Disabled` | AdminLink function is turned off | User disables via Web UI | `Unregistered` (when re-enabled) |
| `Unregistered` | Not registered to AdminLink service | API returns 401; factory reset | `Registered`, `Checking` |
| `Registered` | Successfully registered | API returns 201 or 200 (dev_id_changed=0) | `Unregistered`, `Checking`, `re-registerable` |
| `Checking` | Registration status cannot be determined | API returns other error; no response | `Registered`, `Unregistered` (after retry) |
| `re-registerable` | Factory-reset after registration | API returns 200 (dev_id_changed=1) | `Registered` (after re-registration) |

### AGT.1.3.30 — Issue device registration code

#### `AGT.1.3.31`

The value of the "Device Registration Code" item in the UI is initially set to blank and nothing is displayed until the device registration code is generated. The item name is always displayed.

#### `AGT.1.3.32`

When the "Issue Device Registration Code" button is clicked, the device registration code is generated and displayed in the "Device Registration Code" section of the UI.

Even if the device registration code has already been generated and displayed, it will be generated and the display will be updated when the "Issue Device Registration Code" button is clicked.

#### `AGT.1.3.33`

The device registration code shall be generated by the following calculation method.
- 9 hexadecimal digits (uppercase letters are used for character strings)
- 1-4 digits: Last 4 digits of the device's MAC address
- 5-8 digits: Generation date and time (day/hour/minute), 4 hexadecimal digits. Spaces are filled with zeroes (zero-fill).
- 9 digit: Least significant digit of the value obtained by adding digits 1-8 (hexadecimal number)

ex) At 13:59 on July 30, 2021
30 days -> 30 x 24 hours x 60 minutes = 43200 minutes
13 hours -> 13 x 60 minutes = 780 minutes
59 minutes = 59 minutes
The total above is 44,039 decimal minutes, which is "AC07" when converted to hexadecimal.

ex) If digits 1-8 are E528AC07
E+5+2+8+A+C+0+7 = 3A
From the above formula, it becomes 'A'.

#### `AGT.1.3.34`

Display Device Registration Code by Read-Only or Write-protection.

#### `AGT.1.3.35`

The device doesn't retain the value of the generated "Device Registration Code".
Discard the value, when the browser is closed, or client moves to other screen of Web UI.
