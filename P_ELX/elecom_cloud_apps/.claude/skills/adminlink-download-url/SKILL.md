# SKILL: URL Acquisition API for File Download (2.8)

## Trigger Conditions

Use this SKILL when the user asks to:
- Obtain a pre-signed S3 URL to download a file from the service
- Use a `download_id` received from a cloud-side remote control instruction
- Understand the download URL lifecycle and S3 temporary storage behavior
- Diagnose an error response from this API

---

## API Overview

| Item | Value |
|------|-------|
| URI | `https://api.admin-link.net/v1/devices/{dev_id}/downloadurl/{download_id}` |
| Method | `GET` |
| X-NASAPI-Device-Subkey | MAC address of the device |
| Request Body | None |
| Purpose | Called by the device to obtain the download destination URL for downloading files from the service |

> Both `{dev_id}` and `{download_id}` must be set in the URI path.
> `download_id` is the ID notified from the cloud side to the device at the time of remote control instructions for firmware update, remote setting, and setting restoration. Set the notified ID as-is into the URL.

---

## File Download Workflow Context

This API is **Step 1 of a 2-step download flow**:

```
[Cloud side]
  Remote control instruction → notifies device of download_id

[Device side]
Step 1 — Call THIS API (2.8) with download_id
  → Receive: pre-signed S3 download URL (download_url)

Step 2 — Download file using download_url (GET from S3)
  → File is in TEMPORARY S3 storage

Step 3 — Call 2.9 API (File Download Completion Notification)
  → Notifies service that download is complete
  → Service deletes file from temporary S3
  → If 2.9 API call fails → file remains in temporary S3
     → Auto-deleted after a period per S3 settings
```

---

## Execution Steps

### Step 1 — Obtain the download_id

The `download_id` is **not generated by the device**. It is:
- Notified from the cloud side at the time of remote control instructions
- Used for: firmware update, remote setting, setting restoration
- Must be set into the URI exactly as received — do not modify

---

### Step 2 — Build the Request

No request body required.

| Header | Value |
|--------|-------|
| `X-NASAPI-Device-Subkey` | Representative MAC address of the device |

> No SHA-256 Authorization signature required.

---

### Step 3 — Send the Request

**curl example:**
```bash
DEV_ID="NAS001"
MAC_ADR="AA:BB:CC:DD:EE:FF"
DOWNLOAD_ID="DOWNLOAD_ID_FROM_CLOUD"

curl -X GET "https://api.admin-link.net/v1/devices/${DEV_ID}/downloadurl/${DOWNLOAD_ID}" \
  -H "X-NASAPI-Device-Subkey: ${MAC_ADR}"
```

**Python example:**
```python
import requests

dev_id = "NAS001"
mac_adr = "AA:BB:CC:DD:EE:FF"
download_id = "DOWNLOAD_ID_FROM_CLOUD"

url = f"https://api.admin-link.net/v1/devices/{dev_id}/downloadurl/{download_id}"
headers = {
    "X-NASAPI-Device-Subkey": mac_adr
}

response = requests.get(url, headers=headers)

if response.status_code == 200:
    data = response.json()
    download_url = data["download_url"]
    print("Download URL acquired successfully")
    print(f"Download URL: {download_url}")
    # → Proceed: GET file from download_url, then call 2.9 API
else:
    error = response.json()
    print(f"Error {response.status_code}: [{error['error_id']}] {error['error_msg']}")
```

---

### Step 4 — Handle the Response

#### HTTP Status Codes

| Status | Description | Meaning |
|--------|-------------|---------|
| `200` | OK | URL for downloading successfully obtained |
| `400` | Bad Request | One of the input parameters is not specified or the format is invalid |
| `401` | Unauthorized | Failed to confirm registration by device ID or MAC address (device not registered) |
| `405` | Method Not Allowed | Error in the API invocation method |
| `500` | Internal Server Error | Program error or database error |

---

#### Success (HTTP 200) — Response Body (JSON)

| Key | Required | Description |
|-----|----------|-------------|
| `download_url` | ○ | Pre-signed AWS S3 URL used to download files from the device. Issued by AWS function. Includes a signature to authenticate as a download-only user. **Temporary URL — set to expire in seconds.** S3 download destination is temporary storage — deleted by the 2.9 API call. |

**Pre-signed URL properties:**
```
- Issued by AWS (S3 pre-signed URL)
- Contains embedded signature → download-only permission
- Has a configurable expiry time (in seconds)
- Points to TEMPORARY S3 storage location
- Must be used before expiry
- After successful 2.9 API call → file is deleted from temporary S3
- If 2.9 API call fails → file remains in temporary S3
  → Auto-deleted after a period configured in S3 settings
```

**NextGiga — Configuration Data File Behavior (decided 2025/07/15):**
```
- Configuration data file is downloaded as a SINGLE file
  (Because the download URL is issued for downloading a single file)
- Even if the schema JSON is divided into individual files,
  type number files, etc. → must be combined into a single file
- Even if the schema JSON is divided into model number files, etc.
  → must be combined into a single file
- If it is more convenient for the device side to be separated
  → it is possible to download a single compressed file
- Decided: make it one JSON file (2025/07/15 meeting)
```

### Step 5 — Diagnose Errors

#### HTTP 400 errors

| error_id | Error Message (JP) | Meaning | Fix |
|----------|--------------------|---------|-----|
| `4000` | 必須項目が入力されていません。 | A required field in the request header is not specified. Key Required Error | Ensure `X-NASAPI-Device-Subkey` is present and both `{dev_id}` and `{download_id}` are in the URI |
| `4002` | 必須項目が入力されていません。 | The request parameter item exists, but the value is not specified. Required error for value | Ensure URI path values are non-empty |
| `4004` | 入力可能な桁数を超過しています。 | Number of digits error | Check length of `dev_id` and `download_id` in URI |
| `4005` | 入力された文字種が不正です。 | Character type error | Check for invalid characters in `dev_id` or MAC address |
| `4006` | フォーマットエラーが発生しました。 | MAC address format error | Verify MAC address format (e.g., `AA:BB:CC:DD:EE:FF`) |

#### HTTP 401 errors

| error_id | Error Message (JP) | Meaning | Fix |
|----------|--------------------|---------|-----|
| `4012` | デバイス情報が登録されていません。 | Device information with the same device ID and MAC address is not registered | Verify `dev_id` and MAC address match a registered device |

#### HTTP 405 errors

| error_id | Error Message (JP) | Meaning | Fix |
|----------|--------------------|---------|-----|
| `405` | 呼び出し方法に誤りがあります。 | 405 error occurred | Verify HTTP method is `GET` |

#### HTTP 500 errors

| error_id | Error Message (JP) | Meaning |
|----------|--------------------|---------|
| `5000` | システムエラーが発生しました。 | A 500 error occurred |
| `5001` | システムエラーが発生しました。 | Failed to acquire constant information from the system master |
| `5002` | システムエラーが発生しました。 | Failed to connect to S3 |
| `5003` | システムエラーが発生しました。 | Failed to move the file to the temporary file storage location in S3 |
| `5004` | システムエラーが発生しました。 | Failed to issue URL for download |

## Business Rules Summary (always enforce)

### Call Order Dependency
```
MUST follow this sequence:
  1. Receive download_id from cloud remote control instruction
  2. Call THIS API (2.8) → get download_url
  3. GET file from S3 using download_url (before expiry)
  4. Call 2.9 API with download_id → triggers deletion from temporary S3

Do NOT call this API without a valid download_id from the cloud.
Do NOT skip the 2.9 API after a successful download.
```

### download_id Origin
```
- download_id is ALWAYS provided by the cloud side
- Triggered by remote control instructions for:
    * Firmware update
    * Remote setting
    * Setting restoration
    * NextGiga: additional download targets (no spec change)
- Do NOT generate or guess download_id on the device side
- Set it into the URI exactly as received
```

### download_url Lifecycle
```
- download_url is temporary and expires in seconds
- download_url is download-only (signature embedded)
- Do not cache or reuse across sessions
- If expired → call this API again for a fresh URL
```

### NextGiga Configuration Data File Rule (2025/07/15)
```
Configuration data file format = ONE JSON file
- Combine all schema JSON fragments into a single file before serving
- Alternatively, serve as a single compressed file if separation is needed
- Decision is final as of 2025/07/15 meeting
```

### Authentication
```
dev_id      → URI path
download_id → URI path
MAC address → X-NASAPI-Device-Subkey header
No SHA-256 signature required
Mismatch → 4012
```

---

## Response Principles

1. Always use `download_id` exactly as received from the cloud — never modify it
2. Always call the 2.9 API after a successful download to trigger temporary S3 cleanup
3. Treat `download_url` as short-lived — never reuse across sessions or retry attempts
4. For NextGiga: configuration data must be served as a single JSON or compressed file
5. On 500-series errors, use `error_id` to identify whether failure was in S3 connect (`5002`), file staging (`5003`), or URL issuance (`5004`)
